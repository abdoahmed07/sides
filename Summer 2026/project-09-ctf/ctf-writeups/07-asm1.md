# PicoCTF — asm1 (Reverse Engineering)

**Category:** Reverse Engineering
**Points:** 200

## Background

Before attempting this I spent 2 hours reading the x86 Assembly Guide at cs.virginia.edu. Key things I needed to understand:
- `eax`, `ebx`, `ecx`, `edx` are general-purpose 32-bit registers
- `cmp a, b` sets processor flags by computing `a - b` (doesn't store result)
- `je` = jump if equal (ZF=1), `jle` = jump if less-or-equal, `jg` = jump if greater
- `mov dest, src` moves a value
- `ret` returns from the function with `eax` as the return value

## The challenge

I was given an assembly function and asked: what does `asm1(0x2e0)` return?

```asm
asm1:
  push   ebp
  mov    ebp, esp
  cmp    DWORD PTR [ebp+0x8], 0x3a2    ; compare arg (0x2e0) with 0x3a2
  jg     part_b                         ; jump if arg > 0x3a2 (not taken, 0x2e0 < 0x3a2)
  cmp    DWORD PTR [ebp+0x8], 0x1f1    ; compare arg (0x2e0) with 0x1f1
  jne    part_a                         ; jump if not equal (taken, 0x2e0 != 0x1f1)
  ; ... (skipped)
part_a:
  mov    eax, DWORD PTR [ebp+0x8]      ; eax = 0x2e0
  add    eax, 0x3                       ; eax = 0x2e3
  jmp    end
end:
  pop    ebp
  ret                                   ; return eax = 0x2e3
```

## Tracing the execution

1. `arg = 0x2e0 = 736`
2. `0x2e0 > 0x3a2 (930)?` — No (736 < 930), don't jump
3. `0x2e0 == 0x1f1 (497)?` — No, jump to `part_a`
4. `eax = 0x2e0`, `eax += 3` → `eax = 0x2e3 = 739`
5. Return `739`

Answer: `0x2e3` = `739`

## What I learned

x86 assembly isn't as hard to read as I expected — you just need to understand the basic instruction set and track values through registers. The pattern of `cmp` followed by a conditional jump is how all if/else logic looks in assembly. Tracing execution manually for small functions is completely doable.
